## Prerequisites

Before you begin, ensure you have:

<Steps>
  <Step title="FWCloud API Running">
    The FWCloud API server must be running and accessible. By default, it runs on `https://127.0.0.1:3131`
  </Step>
  <Step title="Valid Credentials">
    You need a valid customer ID, username, and password
  </Step>
  <Step title="HTTP Client">
    Use curl, Postman, or any HTTP client that supports cookies/sessions
  </Step>
</Steps>

## Your First API Call

Let's make your first API call by logging in to the FWCloud API.

### Step 1: Login

The login endpoint authenticates you and creates a session:

<CodeGroup>
```bash curl
curl -k -X POST https://127.0.0.1:3131/user/login \
  -H "Content-Type: application/json" \
  -c cookies.txt \
  -d '{
    "customer": 1,
    "username": "fwcadmin",
    "password": "fwcadmin"
  }'
```

```javascript JavaScript
const response = await fetch('https://127.0.0.1:3131/user/login', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  credentials: 'include', // Important: include cookies
  body: JSON.stringify({
    customer: 1,
    username: 'fwcadmin',
    password: 'fwcadmin'
  })
});

const data = await response.json();
console.log(data);
```

```python Python
import requests

url = 'https://127.0.0.1:3131/user/login'
payload = {
    'customer': 1,
    'username': 'fwcadmin',
    'password': 'fwcadmin'
}

session = requests.Session()
response = session.post(url, json=payload, verify=False)
print(response.json())
```
</CodeGroup>

<Note>
  The `-k` flag in curl disables certificate verification for self-signed certificates. In production, use valid certificates and remove this flag.
</Note>

**Response:**
```json
{
  "user": 1,
  "role": 1,
  "publicKey": "-----BEGIN PGP PUBLIC KEY BLOCK-----\n..."
}
```

The response includes:
- `user`: Your user ID
- `role`: Your role (1 = Admin, 2 = Manager)
- `publicKey`: Server's PGP public key for encrypted communication

<Warning>
  The `-c cookies.txt` flag is crucial! It saves the session cookie which you'll need for all subsequent requests.
</Warning>

### Step 2: Make an Authenticated Request

Now that you're logged in, let's make an authenticated request to keep your session alive:

<CodeGroup>
```bash curl
curl -k -X PUT https://127.0.0.1:3131/ping \
  -H "Content-Type: application/json" \
  -b cookies.txt
```

```javascript JavaScript
// Session is maintained via cookies
const response = await fetch('https://127.0.0.1:3131/ping', {
  method: 'PUT',
  credentials: 'include'
});

const data = await response.json();
console.log(data);
```

```python Python
# Session is maintained automatically
response = session.put('https://127.0.0.1:3131/ping')
print(response.json())
```
</CodeGroup>

**Response:**
```json
{
  "result": "pong"
}
```

### Step 3: Logout

When you're done, properly close your session:

<CodeGroup>
```bash curl
curl -k -X POST https://127.0.0.1:3131/user/logout \
  -b cookies.txt
```

```javascript JavaScript
await fetch('https://127.0.0.1:3131/user/logout', {
  method: 'POST',
  credentials: 'include'
});
```

```python Python
response = session.post('https://127.0.0.1:3131/user/logout')
print(response.status_code)  # 204 No Content
```
</CodeGroup>

**Response:**
```
HTTP/1.1 204 No Content
```

## Understanding Sessions

The FWCloud API uses **session-based authentication**:

<Steps>
  <Step title="Login Creates Session">
    When you login, the server creates a session file and returns a session cookie
  </Step>
  <Step title="Include Cookie in Requests">
    You must include this cookie in all subsequent requests
  </Step>
  <Step title="Session Keepalive">
    Sessions automatically expire after a period of inactivity. Use the `/ping` endpoint to keep sessions alive
  </Step>
  <Step title="Logout Destroys Session">
    Always logout to properly clean up server-side session files
  </Step>
</Steps>

## Common Endpoints

Here are some commonly used endpoints to get you started:

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/user/login` | POST | Authenticate and create session |
| `/user/logout` | POST | Destroy session |
| `/ping` | PUT | Keep session alive (ping/pong) |
| `/config` | GET | Get API configuration |
| `/fwclouds` | POST | Create a new firewall cloud |
| `/version` | GET | Get API version info |

## Error Handling

The API uses standard HTTP status codes:

- `200` - Success
- `204` - Success (No Content)
- `400` - Bad Request (validation error)
- `401` - Unauthorized (authentication failed)
- `404` - Not Found
- `500` - Internal Server Error

**Error Response Example:**
```json
{
  "fwcErr": 1001,
  "msg": "Bad username or password"
}
```

## Session Expiration

Sessions expire after a period of inactivity (configurable via `session.keepalive_ms`):

```json
{
  "fwcErr": 9003,
  "msg": "Session expired"
}
```

<Info>
  If you receive a session expired error, simply login again to create a new session.
</Info>

## Complete Example

Here's a complete example workflow:

```bash
# 1. Login and save session
curl -k -X POST https://127.0.0.1:3131/user/login \
  -H "Content-Type: application/json" \
  -c cookies.txt \
  -d '{"customer":1,"username":"fwcadmin","password":"fwcadmin"}'

# 2. Make authenticated requests
curl -k -X PUT https://127.0.0.1:3131/ping -b cookies.txt

# 3. Get configuration
curl -k -X GET https://127.0.0.1:3131/config -b cookies.txt

# 4. Logout
curl -k -X POST https://127.0.0.1:3131/user/logout -b cookies.txt

# 5. Clean up
rm cookies.txt
```
