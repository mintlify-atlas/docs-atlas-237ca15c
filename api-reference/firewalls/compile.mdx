Policy compilation endpoints transform firewall rules and system configurations into executable scripts ready for deployment.

## Compile Routing Rules

`GET /fwclouds/:fwcloud/firewalls/:firewall/routingRules/compile`

Compiles routing rules for a specific firewall into executable routing table commands.

### Path Parameters

<ParamField path="fwcloud" type="number" required>
  The unique identifier of the FWCloud
</ParamField>

<ParamField path="firewall" type="number" required>
  The unique identifier of the firewall
</ParamField>

### Query Parameters

<ParamField query="rules" type="array">
  Optional array of specific routing rule IDs to compile. If not provided, all rules are compiled.
</ParamField>

### Response

<ResponseField name="script" type="string">
  Compiled shell script with routing commands
</ResponseField>

<ResponseField name="rules" type="array">
  Array of compiled rule objects
</ResponseField>

### Example Request

```bash cURL
curl -X GET "https://api.fwcloud.net/fwclouds/1/firewalls/5/routingRules/compile?rules=10&rules=11" \
  -H "Authorization: Bearer YOUR_API_TOKEN"
```

### Example Response

```json
{
  "script": "#!/bin/bash\n# Routing rules compilation\nip route add 10.0.1.0/24 via 192.168.1.1 dev eth0\nip route add 10.0.2.0/24 via 192.168.1.2 dev eth1\n",
  "rules": [
    {
      "id": 10,
      "command": "ip route add 10.0.1.0/24 via 192.168.1.1 dev eth0",
      "comment": "Route to internal network"
    },
    {
      "id": 11,
      "command": "ip route add 10.0.2.0/24 via 192.168.1.2 dev eth1",
      "comment": "Route to DMZ"
    }
  ]
}
```

## Common Errors

<ResponseField name="404" type="error">
  Not Found - FWCloud, firewall, or rules not found
</ResponseField>

<ResponseField name="403" type="error">
  Forbidden - Insufficient permissions to compile rules
</ResponseField>

<ResponseField name="400" type="error">
  Bad Request - Invalid rule configuration
</ResponseField>

## Compilation Notes

- Compilation validates rules before generating configuration
- Rules are compiled in the order they are defined
- Firewall-specific rules are filtered based on the target firewall
- Generated scripts include comments for documentation
- Compilation does not apply changes to the firewall (use install endpoints for that)
