## Check Policy Script

`POST /fwclouds/:fwcloud/firewalls/:firewall/AIassistant`

Uses AI to analyze a firewall policy script and provide security recommendations, identify potential issues, and suggest improvements.

<Info>
  This endpoint requires OpenAI API configuration. See [AI Assistant Configuration](/api-reference/ai/config) for setup.
</Info>

### Path Parameters

<ParamField path="fwcloud" type="number" required>
  FWCloud ID
</ParamField>

<ParamField path="firewall" type="number" required>
  Firewall ID
</ParamField>

### Request Body

<ParamField body="script" type="string" required>
  The firewall policy script to analyze (bash script for iptables/nftables)
</ParamField>

<ParamField body="prompt" type="string">
  Optional custom prompt or question about the policy
</ParamField>

<ParamField body="compiler" type="string">
  Compiler type: `iptables`, `nftables`, or `vyos`
</ParamField>

### Example Request

```bash cURL
curl -X POST https://api.fwcloud.local/fwclouds/1/firewalls/1/AIassistant \
  -H "Content-Type: application/json" \
  -H "Cookie: session=your-session-cookie" \
  -d '{
    "script": "#!/bin/bash\niptables -P INPUT DROP\niptables -P FORWARD DROP\niptables -A INPUT -p tcp --dport 22 -j ACCEPT\niptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT",
    "compiler": "iptables",
    "prompt": "Review this firewall policy for security issues"
  }'
```

```javascript JavaScript
const policyScript = `#!/bin/bash
iptables -P INPUT DROP
iptables -P FORWARD DROP
iptables -A INPUT -p tcp --dport 22 -j ACCEPT
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT
`;

const response = await fetch(
  'https://api.fwcloud.local/fwclouds/1/firewalls/1/AIassistant',
  {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Cookie': sessionCookie
    },
    body: JSON.stringify({
      script: policyScript,
      compiler: 'iptables',
      prompt: 'Review this firewall policy for security issues'
    })
  }
);

const analysis = await response.json();
console.log('AI Analysis:', analysis.response);
```

## AI Analysis Features

The AI assistant can:

- ✅ Identify overly permissive rules
- ✅ Detect missing stateful connection tracking
- ✅ Find rules that may conflict
- ✅ Suggest rate limiting for exposed services
- ✅ Recommend logging improvements
- ✅ Identify missing egress filtering
- ✅ Detect dangerous ACCEPT ALL rules
- ✅ Suggest DDoS mitigation techniques
- ✅ Check for proper default policies
- ✅ Recommend redundant rule removal

## Integration Example

```javascript
// Compile policy and analyze with AI
async function reviewPolicy(fwcloudId, firewallId) {
  // 1. Get policy script
  const policyResponse = await fetch(
    `https://api.fwcloud.local/fwclouds/${fwcloudId}/firewalls/${firewallId}/policyRules/read`,
    { method: 'GET', headers: { Cookie: sessionCookie } }
  );
  const script = await policyResponse.text();
  
  // 2. Analyze with AI
  const analysisResponse = await fetch(
    `https://api.fwcloud.local/fwclouds/${fwcloudId}/firewalls/${firewallId}/AIassistant`,
    {
      method: 'POST',
      headers: { 'Content-Type': 'application/json', Cookie: sessionCookie },
      body: JSON.stringify({
        script,
        compiler: 'iptables',
        prompt: 'Security audit and best practices review'
      })
    }
  );
  
  const analysis = await analysisResponse.json();
  console.log(analysis.response);
  console.log(`Cost: $${analysis.cost_estimate.toFixed(4)}`);
}
```

## Cost Management

<Warning>
  Each AI request consumes OpenAI API tokens and incurs costs. Typical analysis:
  - GPT-4: $0.01-0.05 per request
  - GPT-3.5-turbo: $0.001-0.005 per request
  
  Monitor usage to control costs.
</Warning>

Cost-saving tips:

1. Use `gpt-3.5-turbo` for routine checks
2. Use `gpt-4` only for complex analysis
3. Batch multiple policies into one request
4. Cache results for unchanged policies
