## Check Compiled Rules

`POST /fwclouds/:fwcloud/firewalls/:firewall/AIassistant/rules`

Uses AI to analyze compiled firewall rules (individual iptables/nftables commands) for security issues, redundancies, and optimization opportunities.

<Info>
  This endpoint requires OpenAI API configuration. See [AI Assistant Configuration](/api-reference/ai/config) for setup.
</Info>

### Path Parameters

<ParamField path="fwcloud" type="number" required>
  FWCloud ID
</ParamField>

<ParamField path="firewall" type="number" required>
  Firewall ID
</ParamField>

### Request Body

<ParamField body="rules" type="array" required>
  Array of compiled rule objects to analyze
</ParamField>

<ParamField body="rules[].rule" type="string" required>
  The compiled rule command (e.g., iptables command)
</ParamField>

<ParamField body="rules[].id" type="number">
  Policy rule ID for reference
</ParamField>

<ParamField body="prompt" type="string">
  Optional custom prompt or specific question
</ParamField>

### Example Request

```bash cURL
curl -X POST https://api.fwcloud.local/fwclouds/1/firewalls/1/AIassistant/rules \
  -H "Content-Type: application/json" \
  -H "Cookie: session=your-session-cookie" \
  -d '{
    "rules": [
      {
        "id": 1,
        "rule": "iptables -A INPUT -p tcp --dport 22 -j ACCEPT"
      },
      {
        "id": 2,
        "rule": "iptables -A INPUT -p tcp -s 0.0.0.0/0 --dport 80 -j ACCEPT"
      },
      {
        "id": 3,
        "rule": "iptables -A INPUT -p tcp --dport 443 -j ACCEPT"
      }
    ],
    "prompt": "Analyze these rules for security issues"
  }'
```

```javascript JavaScript
const rules = [
  { id: 1, rule: 'iptables -A INPUT -p tcp --dport 22 -j ACCEPT' },

  { id: 3, rule: 'iptables -A INPUT -p tcp --dport 443 -j ACCEPT' }

  'https://api.fwcloud.local/fwclouds/1/firewalls/1/AIassistant/rules',

      'Cookie': sessionCookie
    },

const analysis = await response.json();
console.log(analysis.response);
```

## Analysis Types

<Tabs>
  <Tab title="Security Analysis">
    Identify security vulnerabilities in rules:
    
    ```json
    {
      "prompt": "Identify security vulnerabilities and risky configurations"
    }
    ```
    
    AI checks for:
    - Overly permissive source addresses
    - Missing rate limiting
    - Dangerous port exposures
    - Missing connection state tracking
  </Tab>
  
  <Tab title="Redundancy Check">
    Find redundant or conflicting rules:
    
    ```json
    {
      "prompt": "Find redundant rules and optimization opportunities"
    }
    ```
    
    AI identifies:
    - Duplicate rules
    - Rules that can be combined
    - Rules that never match (shadowed)
    - Unnecessary explicit specifications
  </Tab>
  
  <Tab title="Order Optimization">
    Analyze rule ordering:
    
    ```json
    {
      "prompt": "Are these rules in optimal order for performance?"
    }
    ```
    
    AI suggests:
    - Moving frequently-matched rules earlier
    - Grouping related rules
    - Optimizing chain traversal

  <Tab title="Compliance Check">

## Workflow Integration

```javascript
// Analyze policy rules during policy compilation
async function compileAndAnalyze(fwcloudId, firewallId) {
  // 1. Compile policy
  const compileRes = await fetch(
    `https://api.fwcloud.local/fwclouds/${fwcloudId}/firewalls/${firewallId}/policyRules/read`,
    { headers: { Cookie: sessionCookie } }
  );
  const script = await compileRes.text();
  
  // 2. Extract individual rules
  const rules = script
    .split('\n')
    .filter(line => line.startsWith('iptables'))
    .map((rule, idx) => ({ id: idx + 1, rule }));
  
  // 3. Analyze rules
  const analysisRes = await fetch(
    `https://api.fwcloud.local/fwclouds/${fwcloudId}/firewalls/${firewallId}/AIassistant/rules`,
    {
      method: 'POST',
      headers: { 'Content-Type': 'application/json', Cookie: sessionCookie },
      body: JSON.stringify({
        rules,
        prompt: 'Security and optimization analysis'
      })
    }
  );
  
  const analysis = await analysisRes.json();
  
  // 4. Report critical issues
  const critical = analysis.rule_analysis.filter(r => r.severity === 'critical');
  if (critical.length > 0) {
    console.error('CRITICAL ISSUES FOUND:');
    critical.forEach(r => {
      console.error(`Rule ${r.rule_id}:`, r.findings);
    });
  }
  
  return analysis;
}
```

## Batch Analysis

For large policies, batch rules into groups:

```javascript
const BATCH_SIZE = 20;

for (let i = 0; i < rules.length; i += BATCH_SIZE) {
  const batch = rules.slice(i, i + BATCH_SIZE);
  const analysis = await analyzeRules(batch);
  // Process analysis results
}
```

<Tip>
  Analyzing 10-20 rules per request provides good balance between cost and completeness.
</Tip>
