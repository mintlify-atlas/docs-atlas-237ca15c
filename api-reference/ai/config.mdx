## Get AI Configuration

`GET /aiassistant`

Retrieves the current AI assistant configuration including API settings and model preferences.

### Response

<ResponseField name="enabled" type="boolean">
  Whether AI assistant is enabled
</ResponseField>

<ResponseField name="api_key_configured" type="boolean">
  Whether OpenAI API key is set (key value is never returned)
</ResponseField>

<ResponseField name="model" type="string">
  OpenAI model being used (e.g., `gpt-4`, `gpt-3.5-turbo`)
</ResponseField>

<ResponseField name="max_tokens" type="number">
  Maximum tokens per request
</ResponseField>

<ResponseField name="temperature" type="number">
  Model temperature (0.0-2.0)
</ResponseField>

```json
{
  "enabled": true,
  "api_key_configured": true,
  "model": "gpt-4",
  "max_tokens": 2000,
  "temperature": 0.3
}
```

## Update AI Configuration

`PUT /aiassistant`

Updates AI assistant settings. Requires admin privileges.

### Request Body

<ParamField body="enabled" type="boolean">
  Enable or disable AI assistant
</ParamField>

<ParamField body="api_key" type="string">
  OpenAI API key (stored encrypted)
</ParamField>

<ParamField body="model" type="string">
  OpenAI model to use:
  - `gpt-4` - Most capable, higher cost
  - `gpt-4-turbo` - Fast GPT-4, lower cost
  - `gpt-3.5-turbo` - Fast, cost-effective
</ParamField>

<ParamField body="max_tokens" type="number">
  Maximum tokens per request (100-4000)
</ParamField>

<ParamField body="temperature" type="number">
  Model temperature (0.0-2.0):
  - Lower (0.0-0.5): More focused, deterministic
  - Higher (0.5-1.0): More creative
</ParamField>

### Example Request

```bash cURL
curl -X PUT https://api.fwcloud.local/aiassistant \
  -H "Content-Type: application/json" \
  -H "Cookie: session=admin-session-cookie" \
  -d '{
    "enabled": true,
    "api_key": "sk-proj-...",
    "model": "gpt-4",
    "max_tokens": 2000,
    "temperature": 0.3
  }'
```

```javascript JavaScript
const response = await fetch('https://api.fwcloud.local/aiassistant', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
    'Cookie': adminSessionCookie
  },
  body: JSON.stringify({
    enabled: true,
    api_key: 'sk-proj-...',
    model: 'gpt-4',
    max_tokens: 2000,
    temperature: 0.3
  })
});

const config = await response.json();
console.log('AI configured:', config);
```

### Response

Returns the updated configuration (without the API key).

## Delete AI Configuration

`DELETE /aiassistant`

Removes AI assistant configuration and deletes the stored API key.

```bash
curl -X DELETE https://api.fwcloud.local/aiassistant \
  -H "Cookie: session=admin-session-cookie"
```

## Model Selection Guide

<Tabs>
  <Tab title="GPT-4">
    **Best for:** Complex security analysis, detailed reviews
    
    - **Cost:** ~$0.03 per request
    - **Speed:** 5-10 seconds
    - **Quality:** Highest
    - **Context:** 8K tokens
    
    **Use when:**
    - Performing security audits
    - Analyzing complex policies
    - Need detailed explanations
    - Compliance checking
  </Tab>
  
  <Tab title="GPT-4 Turbo">
    **Best for:** Balance of quality and cost
    
    - **Cost:** ~$0.01 per request
    - **Speed:** 2-5 seconds
    - **Quality:** Very high
    - **Context:** 128K tokens
    
    **Use when:**
    - General policy analysis
    - Daily security checks
    - Large policy reviews
    - Good balance needed
  </Tab>
  
  <Tab title="GPT-3.5 Turbo">
    **Best for:** Quick checks, high volume
    
    - **Cost:** ~$0.002 per request
    - **Speed:** 1-3 seconds
    - **Quality:** Good
    - **Context:** 4K tokens
    
    **Use when:**
    - Quick syntax checks
    - Simple rule analysis
    - High request volume
    - Cost is primary concern
  </Tab>
</Tabs>

## Temperature Settings

Temperature controls response randomness:

<CardGroup cols={3}>
  <Card title="Low (0.0-0.3)">
    **Recommended for security analysis**
    
    - Focused, consistent
    - Follows established patterns
    - Predictable recommendations
  </Card>
  
  <Card title="Medium (0.4-0.7)">
    Balanced creativity and consistency
    
    - Some variation
    - Good for general analysis
  </Card>
  
  <Card title="High (0.8-1.0)">
    Creative, diverse responses
    
    - More variation
    - Less predictable
    - Not recommended for security
  </Card>
</CardGroup>

<Tip>
  For firewall policy analysis, use temperature **0.3** for consistent, focused security recommendations.
</Tip>

## API Key Security

<Warning>
  The OpenAI API key is sensitive. It is:
  - Stored encrypted in the database
  - Never returned in API responses
  - Only accessible to admin users
  - Transmitted over HTTPS only
</Warning>

**Getting an OpenAI API Key:**

<Steps>
  <Step title="Create OpenAI Account">
    Sign up at [platform.openai.com](https://platform.openai.com)
  </Step>
  
  <Step title="Add Payment Method">
    Add a payment method in account billing settings
  </Step>
  
  <Step title="Generate API Key">
    Go to API Keys section and create a new key
  </Step>
  
  <Step title="Set Usage Limits">
    Configure monthly spending limits to control costs
  </Step>
</Steps>

## Cost Management

**Estimated Costs (per 1000 requests):**

| Model | Input | Output | Total per 1K |
|-------|-------|--------|-------------|
| GPT-4 | $0.03/1K tokens | $0.06/1K tokens | ~$30-50 |
| GPT-4 Turbo | $0.01/1K tokens | $0.03/1K tokens | ~$10-20 |
| GPT-3.5 Turbo | $0.0005/1K tokens | $0.0015/1K tokens | ~$2-5 |

**Cost-saving strategies:**

1. Use GPT-3.5 for routine checks
2. Limit max_tokens to actual needs
3. Cache results for unchanged policies
4. Batch multiple policies into one request
5. Set OpenAI usage limits

## Configuration via Environment

You can also configure AI settings via environment variables:

```bash .env
AI_ASSISTANT_ENABLED=true
OPENAI_API_KEY=sk-proj-...
OPENAI_MODEL=gpt-4
OPENAI_MAX_TOKENS=2000
OPENAI_TEMPERATURE=0.3
```

API settings override environment variables.

## Testing Configuration

Test AI configuration with a simple request:

```bash
curl -X POST https://api.fwcloud.local/fwclouds/1/firewalls/1/AIassistant \
  -d '{
    "script": "iptables -A INPUT -j ACCEPT",
    "prompt": "Is this rule secure?"
  }'
```

Expected response:

```json
{
  "response": "No, this rule is extremely insecure. It accepts all input traffic without any filtering...",
  "model": "gpt-4",
  "tokens_used": 67
}
```

## Error Responses

<Accordion title="403 Forbidden">
  Only admin users can modify AI configuration.
  
  ```json
  {
    "message": "Admin privileges required"
  }
  ```
</Accordion>

<Accordion title="400 Bad Request">
  Invalid API key or model name.
  
  ```json
  {
    "message": "Invalid OpenAI API key",
    "details": "Authentication failed"
  }
  ```
</Accordion>

<Accordion title="402 Payment Required">
  OpenAI account has insufficient credits.
  
  ```json
  {
    "message": "OpenAI API quota exceeded",
    "details": "Add credits to your OpenAI account"
  }
  ```
</Accordion>
