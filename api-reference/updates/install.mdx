## Install API Update

`PUT /updates/api`

Installs the latest FWCloud API update. The API server will restart automatically after installation.

<Warning>
  **Server Restart Required**
  
  Installing updates will:
  - Terminate all active sessions
  - Restart the API server
  - Cause 1-2 minutes of downtime
  
  Always schedule updates during maintenance windows.
</Warning>

### Request Body

<ParamField body="version" type="string">
  Specific version to install (optional, defaults to latest)
</ParamField>

<ParamField body="pre_update_backup" type="boolean" default="true">
  Create backup before updating (highly recommended)
</ParamField>

### Example Request

```bash cURL
curl -X PUT https://api.fwcloud.local/updates/api \
  -H "Content-Type: application/json" \
  -H "Cookie: session=admin-session-cookie" \
  -d '{
    "version": "2.4.0",
    "pre_update_backup": true
  }'
```

```javascript JavaScript
const response = await fetch('https://api.fwcloud.local/updates/api', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
    'Cookie': adminSessionCookie
  },
  body: JSON.stringify({
    version: '2.4.0',
    pre_update_backup: true
  })
});

const result = await response.json();
console.log('Update started:', result);
```

### Response

<ResponseField name="status" type="string">
  Update status: `started`, `in_progress`, `completed`, or `failed`
</ResponseField>

<ResponseField name="from_version" type="string">
  Version being upgraded from
</ResponseField>

<ResponseField name="to_version" type="string">
  Version being upgraded to
</ResponseField>

<ResponseField name="backup_id" type="number">
  ID of pre-update backup (if created)
</ResponseField>

<ResponseField name="estimated_downtime" type="number">
  Estimated downtime in seconds
</ResponseField>

```json
{
  "status": "started",
  "from_version": "2.3.1",
  "to_version": "2.4.0",
  "backup_id": 47,
  "estimated_downtime": 90
}
```

## Install UI Update

`PUT /updates/ui`

Installs the latest FWCloud UI update. This updates the web interface.

```bash
curl -X PUT https://api.fwcloud.local/updates/ui \
  -H "Cookie: session=admin-session-cookie"
```

<Info>
  UI updates typically do not require API server restart. Users may need to refresh their browser.
</Info>

## Update Process

The update process follows these steps:

<Steps>
  <Step title="Pre-update Backup">
    If enabled, creates a database backup before proceeding.
    
    ```json
    {
      "backup_id": 47,
      "filename": "backup-20240304-153000.sql.gz"
    }
    ```
  </Step>
  
  <Step title="Download Update">
    Downloads the update package from the FWCloud repository.
    
    Progress is reported via WebSocket events (channel: `updates`).
  </Step>
  
  <Step title="Verify Package">
    Verifies package integrity and signature.
  </Step>
  
  <Step title="Run Database Migrations">
    Applies any database schema changes:
    
    ```bash
    node fwcli migration:run
    ```
  </Step>
  
  <Step title="Install Files">
    Replaces application files with new versions.
  </Step>
  
  <Step title="Restart Server">
    Restarts the API server to load new code.
    
    Downtime: 30-90 seconds
  </Step>
  
  <Step title="Post-Update Tasks">
    Runs any post-update scripts or data migrations.
  </Step>
</Steps>

## Progress Tracking

Monitor update progress via WebSocket:

```javascript
import io from 'socket.io-client';

const socket = io('https://api.fwcloud.local', {
  extraHeaders: { Cookie: sessionCookie }
});

socket.on('connect', () => {
  socket.emit('join', { channel: 'updates' });
});

socket.on('update:progress', (data) => {
  console.log(`Update progress: ${data.percent}%`);
  console.log(`Step: ${data.step}`);
});

socket.on('update:completed', (data) => {
  console.log('Update completed:', data);
});

socket.on('update:error', (error) => {
  console.error('Update failed:', error);
});
```

## Rollback on Failure

If an update fails:

1. The system automatically attempts to restore from the pre-update backup
2. If restoration fails, manual intervention is required
3. Use the backup ID from the update response to restore manually

```bash
# Manual rollback
curl -X POST https://api.fwcloud.local/backups/47/restore \
  -d '{"confirm": true, "pre_restore_backup": false}'
```

## Update Strategies

<Tabs>
  <Tab title="Automatic Updates">
    Enable automatic updates for patch versions:
    
    ```bash
    # Auto-install security and patch updates
    crontab -e
    ```
    
    ```cron
    # Check and install patch updates daily at 3 AM
    0 3 * * * /opt/fwcloud-api/bin/fwcloud-update --auto-patch
    ```
  </Tab>
  
  <Tab title="Scheduled Maintenance">
    Schedule updates during maintenance windows:
    
    ```javascript
    // Schedule update for next Sunday at 2 AM
    const updateTime = new Date();
    updateTime.setDate(updateTime.getDate() + (7 - updateTime.getDay()));
    updateTime.setHours(2, 0, 0);
    
    scheduleUpdate(updateTime, '2.4.0');
    ```
  </Tab>
  
  <Tab title="Staged Rollout">
    Update in stages:
    1. Update staging environment
    2. Test for 24-48 hours
    3. Update production during maintenance window
  </Tab>
</Tabs>

## Post-Update Verification

After updating, verify the installation:

```bash
# Check version
curl https://api.fwcloud.local/version

# Test API health
curl https://api.fwcloud.local/ping

# Verify database migrations
node fwcli migration:status

# Check logs for errors
tail -f /var/log/fwcloud-api/api.log
```

## Error Responses

<Accordion title="503 Service Unavailable">
  Update already in progress.
  
  ```json
  {
    "message": "Update already in progress",
    "current_update": {
      "version": "2.4.0",
      "progress": 45
    }
  }
  ```
</Accordion>

<Accordion title="400 Bad Request">
  Invalid version or version already installed.
  
  ```json
  {
    "message": "Version 2.3.1 is already installed"
  }
  ```
</Accordion>

<Accordion title="500 Internal Server Error">
  Update failed. Check logs and backup status.
  
  ```json
  {
    "message": "Update failed",
    "error": "Database migration error",
    "backup_id": 47,
    "rollback_status": "succeeded"
  }
  ```
</Accordion>

## Maintenance Mode

During updates, the API enters maintenance mode:

- All API requests return `503 Service Unavailable`
- WebSocket connections are closed
- Scheduled tasks are paused
- A maintenance page is displayed

## Best Practices

<Steps>
  <Step title="Always Create Backup">
    Never disable `pre_update_backup` unless you have a very recent manual backup.
  </Step>
  
  <Step title="Schedule During Off-Hours">
    Update when user activity is lowest (typically 2-4 AM local time).
  </Step>
  
  <Step title="Notify Users">
    Send advance notice to users about scheduled maintenance:
    
    - 1 week before: Initial notice
    - 1 day before: Reminder
    - 1 hour before: Final warning
  </Step>
  
  <Step title="Test in Staging">
    Always test updates in a staging environment first.
  </Step>
  
  <Step title="Monitor Closely">
    Watch logs and metrics closely for 24 hours after updating.
  </Step>
</Steps>
