## Restore Snapshot

`POST /fwclouds/:fwcloud/snapshots/:snapshot/restore`

Restores a FWCloud to a previous configuration state from a snapshot. This operation replaces the current configuration with the snapshot data.

<Warning>
  **Configuration Replacement**
  
  Restoring a snapshot will:
  - Replace current firewall configurations
  - Replace all policy rules
  - Replace VPN configurations
  - Replace routing tables and rules
  
  This operation can be undone by creating a new snapshot before restoring or by restoring a different snapshot.
</Warning>

### Path Parameters

<ParamField path="fwcloud" type="number" required>
  FWCloud ID
</ParamField>

<ParamField path="snapshot" type="number" required>
  Snapshot ID to restore
</ParamField>

### Request Body

<ParamField body="pre_restore_snapshot" type="boolean" default="true">
  Create a snapshot of the current state before restoring (recommended)
</ParamField>

<ParamField body="pre_restore_snapshot_name" type="string">
  Custom name for the pre-restore snapshot (optional)
</ParamField>

### Example Request

```bash cURL
curl -X POST https://api.fwcloud.local/fwclouds/1/snapshots/15/restore \
  -H "Content-Type: application/json" \
  -H "Cookie: session=your-session-cookie" \
  -d '{
    "pre_restore_snapshot": true,
    "pre_restore_snapshot_name": "auto-before-restore-2024-03-04"
  }'
```

```javascript JavaScript
const response = await fetch(
  'https://api.fwcloud.local/fwclouds/1/snapshots/15/restore',
  {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Cookie': sessionCookie
    },
    body: JSON.stringify({
      pre_restore_snapshot: true,
      pre_restore_snapshot_name: 'auto-before-restore-2024-03-04'
    })
  }
);

const result = await response.json();
console.log('Snapshot restored:', result);
```

```python Python
import requests

response = requests.post(
    'https://api.fwcloud.local/fwclouds/1/snapshots/15/restore',
    json={
        'pre_restore_snapshot': True,
        'pre_restore_snapshot_name': 'auto-before-restore-2024-03-04'
    },
    cookies={'session': session_cookie}
)

result = response.json()
print(f"Snapshot restored: {result}")
```

## Restore Process

The snapshot restore follows these steps:

<Steps>
  <Step title="Pre-restore Snapshot (Optional)">
    If enabled, creates a snapshot of the current configuration for safety.
    
    ```json
    {
      "name": "auto-before-restore-2024-03-04",
      "comment": "Automatic snapshot before restoring snapshot 15"
    }
    ```
  </Step>
  
  <Step title="Validation">
    Validates that:
    - Snapshot exists and belongs to the correct FWCloud
    - Snapshot data is valid and not corrupted
    - User has permission to restore
  </Step>
  
  <Step title="Configuration Replacement">
    Replaces current configuration:
    - Updates firewall settings
    - Replaces policy rules
    - Updates VPN configurations
    - Replaces routing tables
    - Updates system service configs
  </Step>
  
  <Step title="Verification">
    Verifies that:
    - All data was restored correctly
    - Relationships are intact
    - No orphaned records exist
  </Step>
</Steps>

## Rollback Example

Quickly rollback a problematic change:

```bash
# 1. Made a change that broke something
# 2. List snapshots to find last good config
curl https://api.fwcloud.local/fwclouds/1/snapshots

# 3. Restore to last known good snapshot
curl -X POST https://api.fwcloud.local/fwclouds/1/snapshots/15/restore \
  -d '{"pre_restore_snapshot": true}'

# 4. Verify configuration
curl https://api.fwcloud.local/fwclouds/1

# 5. Recompile and deploy policies
curl -X POST https://api.fwcloud.local/fwclouds/1/firewalls/1/policyRules/download \
  -d '{"compiler": "iptables", "install": true}'
```

## Differences from Backup Restore

| Aspect | Snapshot Restore | Backup Restore |
|--------|-----------------|----------------|
| **Scope** | Single FWCloud | Entire database |
| **Impact** | Non-destructive to other FWClouds | Replaces all data |
| **Speed** | Fast (seconds) | Slower (minutes) |
| **Server Restart** | Not required | Required |
| **User Sessions** | Unaffected | Terminated |
| **Use Case** | Undo configuration changes | Disaster recovery |

<Tip>
  Use **snapshot restore** for rolling back configuration changes. Use **backup restore** only for complete disaster recovery.
</Tip>

## Error Responses

<Accordion title="404 Not Found">
  Snapshot or FWCloud does not exist.
  
  ```json
  {
    "message": "Snapshot not found",
    "snapshot_id": 99
  }
  ```
</Accordion>

<Accordion title="400 Bad Request">
  Snapshot data is corrupted or invalid.
  
  ```json
  {
    "message": "Invalid snapshot data",
    "details": "Snapshot file is corrupted"
  }
  ```
</Accordion>

<Accordion title="403 Forbidden">
  User does not have permission to restore snapshots.
  
  ```json
  {
    "message": "Access denied"
  }
  ```
</Accordion>

<Accordion title="409 Conflict">
  FWCloud is currently locked by another user.
  
  ```json
  {
    "message": "FWCloud is locked",
    "locked_by": "admin",
    "locked_at": "2024-03-04T15:00:00.000Z"
  }
  ```
</Accordion>

## Best Practices

<Steps>
  <Step title="Always Create Pre-restore Snapshot">
    Unless you have a very recent snapshot, always enable `pre_restore_snapshot`. The small time cost provides an important safety net.
  </Step>
  
  <Step title="Verify Before Deploying">
    After restoring a snapshot:
    1. Review the configuration in the UI
    2. Compile policies to check for errors
    3. Test in a non-production firewall first (if possible)
    4. Deploy to production firewalls
  </Step>
  
  <Step title="Document Restores">
    Keep a log of snapshot restores:
    - When restored
    - Why restored
    - Which snapshot was used
    - Who performed the restore
  </Step>
  
  <Step title="Clean Up Test Snapshots">
    After successful restore, remove temporary pre-restore snapshots to save storage.
  </Step>
</Steps>

## Compliance and Auditing

Snapshot restores are logged in the audit trail:

```sql
SELECT * FROM audit_log 
WHERE action = 'snapshot.restore' 
AND entity_id = 1 
ORDER BY created_at DESC;
```

This provides a complete history of configuration changes for compliance auditing.
