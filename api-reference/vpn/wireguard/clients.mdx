</ResponseField>

  Endpoint address for this peer
</ResponseField>

## Update Client Peer Options

<api method="PUT" endpoint="/api/fwclouds/{fwcloud}/firewalls/{firewall}/vpn/wireguards/{wireguard}/clients/options">
  Updates peer-specific options for a WireGuard client
</api>

### Body Parameters

<ParamField body="fwcloud" type="number" required>
  FWCloud ID
</ParamField>

<ParamField body="firewall" type="number" required>
  Firewall ID
</ParamField>

<ParamField body="wireguard" type="number" required>
  Server configuration ID
</ParamField>

<ParamField body="wireguard_cli" type="number" required>
  Client configuration ID
</ParamField>

<ParamField body="options" type="array" required>
  Array of peer option objects with scope 3 (client-specific)
  <Expandable title="Option properties">
    <ParamField body="name" type="string" required>
      Option name (e.g., "AllowedIPs", "PersistentKeepalive")
    </ParamField>
    <ParamField body="arg" type="string">
      Option value
    </ParamField>
    <ParamField body="comment" type="string">
      Optional comment
    </ParamField>
  </Expandable>
</ParamField>

### Response

Returns HTTP 204 (No Content) on success.

<ResponseExample>
```bash cURL
curl -X PUT \
  https://api.fwcloud.net/api/fwclouds/1/firewalls/5/vpn/wireguards/42/clients/options \
  -H 'Content-Type: application/json' \
  -d '{
    "fwcloud": 1,
    "firewall": 5,
    "wireguard": 42,
    "wireguard_cli": 43,
    "options": [
      {
        "name": "AllowedIPs",
        "arg": "10.0.0.2/32,192.168.100.0/24"
      },
      {
        "name": "PersistentKeepalive",
        "arg": "30"
      }
    ]
  }'
```
</ResponseExample>

## Notes

- Client peer options are stored with scope 3 to distinguish them from server and interface options
- Updating client options marks the server configuration as modified (requires reinstallation)
- Public keys are PGP encrypted for secure transmission
- The `AllowedIPs` option on the server side determines which traffic the server will accept from this client
- Changes to peer options require a server configuration reload to take effect

## Client Option Scopes

- **Scope 1**: Server interface options
- **Scope 2**: Server peer options
- **Scope 3**: Client-specific peer options (set by these endpoints)

## Workflow

1. Create WireGuard server configuration
2. Create client configurations (one per peer)
3. Configure client-specific peer options (AllowedIPs, keepalive, etc.)
4. Install/reinstall server configuration to apply changes
5. Generate client installers for distribution
