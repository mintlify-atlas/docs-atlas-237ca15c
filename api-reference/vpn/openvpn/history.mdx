---
title: "OpenVPN Status History"
api: "GET /api/fwclouds/{fwcloud}/firewalls/{firewall}/openvpn/{openvpn}/history"
description: "Retrieve OpenVPN connection status history with filtering options"
---

Retrieves historical connection status data for an OpenVPN server, including client connections, disconnections, and connection duration. Supports filtering by time range, client name, and IP address.

## Path Parameters

<ParamField path="fwcloud" type="number" required>
  FWCloud ID
</ParamField>

<ParamField path="firewall" type="number" required>
  Firewall ID
</ParamField>

<ParamField path="openvpn" type="number" required>
  OpenVPN server configuration ID
</ParamField>

## Query Parameters

<ParamField query="starts_at" type="number">
  Unix timestamp (milliseconds) for the start of the time range. Defaults to epoch (0) if not provided.
</ParamField>

<ParamField query="ends_at" type="number">
  Unix timestamp (milliseconds) for the end of the time range. Defaults to current time if not provided.
</ParamField>

<ParamField query="name" type="string">
  Filter results by client common name (CN)
</ParamField>

<ParamField query="address" type="string">
  Filter results by client IP address. Must be a valid IP address format.
</ParamField>

  Array of connection history records

    <ResponseField name="openvpn_id" type="number">

    <ResponseField name="name" type="string">

    <ResponseField name="address" type="string">

## Notes

- Only works with OpenVPN server configurations (not client configurations)
- The server must have status logging enabled and properly configured
- History data is stored in the database by the OpenVPN status monitoring service
- Timestamps use Unix epoch in milliseconds for query parameters
- Returns HTTP 200 with results array (may be empty if no records match)
- Returns HTTP 404 if the OpenVPN server configuration is not found

## Authorization

Requires OpenVPNPolicy.history authorization for the specified OpenVPN server configuration.
