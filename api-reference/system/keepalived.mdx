## Create Keepalived Rule

<CodeGroup>
```bash POST /fwclouds/:fwcloud/firewalls/:firewall/system/keepalivedRules
curl -X POST "https://api.example.com/fwclouds/1/firewalls/1/system/keepalivedRules" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "interface": "eth0",
    "virtual_router_id": 51,
    "priority": 100,
    "virtual_ips": ["192.168.1.100/24"],
    "state": "MASTER",
    "authentication_password": "secret123",
    "comment": "VRRP instance for HA",
    "active": true
  }'
```
</CodeGroup>

**Request Body**
- `interface` (string, optional) - Network interface for VRRP
- `virtual_router_id` (integer, optional) - VRRP instance ID (1-255)
- `priority` (integer, optional) - VRRP priority (1-254)
- `virtual_ips` (array, optional) - Virtual IP addresses
- `state` (string, optional) - Initial state: "MASTER" or "BACKUP"
- `authentication_password` (string, optional) - VRRP authentication password
- `comment` (string, optional) - Description
- `active` (boolean, optional) - Whether rule is enabled
- `group` (integer, optional) - Keepalived group ID

## Install Keepalived Configuration

<CodeGroup>
```bash PUT /fwclouds/:fwcloud/firewalls/:firewall/system/keepalivedRules/install
curl -X PUT "https://api.example.com/fwclouds/1/firewalls/1/system/keepalivedRules/install" \
  -H "Authorization: Bearer YOUR_TOKEN"
```
</CodeGroup>

Compiles and installs Keepalived configuration to the firewall.

This endpoint:
1. Compiles all Keepalived rules into keepalived.conf
2. Uploads the configuration to `/etc/keepalived/keepalived.conf`
3. Validates the configuration
4. Emits progress events via WebSocket

**Response Events (WebSocket)**
```json
{
  "status": "start",
  "message": "Installing Keepalived configuration"
}
```
