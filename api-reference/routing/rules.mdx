## Create Routing Rule

<CodeGroup>
```bash POST /fwclouds/:fwcloud/firewalls/:firewall/routingRules
curl -X POST "https://api.example.com/fwclouds/1/firewalls/1/routingRules" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "from": "10.0.0.0/24",
    "to": "192.168.0.0/16",
    "routingTableId": 100,
    "priority": 1000,
    "comment": "Route traffic from internal network",
    "active": true
  }'
```
</CodeGroup>

**Request Body**
- `from` (string, optional) - Source network (CIDR notation)
- `to` (string, optional) - Destination network (CIDR notation)
- `routingTableId` (integer, required) - Target routing table ID
- `priority` (integer, optional) - Rule priority (lower = higher priority)
- `comment` (string, optional) - Description
- `active` (boolean, optional) - Whether rule is enabled

## Copy Rules

<CodeGroup>
```bash POST /fwclouds/:fwcloud/firewalls/:firewall/routingRules/copy
curl -X POST "https://api.example.com/fwclouds/1/firewalls/1/routingRules/copy" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "rules": ["1", "2"],
    "to": 5,
    "offset": "below"
  }'
```
</CodeGroup>

Copies routing rules to a new position.

**Request Body**
- `rules` (array, required) - Rule IDs to copy
- `to` (integer, required) - Target rule ID to position relative to
- `offset` (string, required) - Position: "above" or "below"
